ROS2ベース自律走行：車線認識エンジンとキャリブレーション
本リポジトリには、卒業課題として取り組んだROS2ベースの自律走行システムのうち、私が主導的に担当した「車線認識エンジン（usb_camパッケージ）」とその精度向上のためのツール群を公開しています。

技術スタック (Tech Stack)
Environment: ROS2 (Humble), Ubuntu 22.04

Languages: C++, Python, MATLAB

Library: OpenCV

主要構成 (Project Structure)
本リポジトリは、全システムの中でも技術的変別力が最も高い「認識および補正アルゴリズム」を中心に構成されています。

1. 車線認識エンジンの核心 (src/usb_cam/src/lane_cam.cpp)
プロジェクトのメインロジックが実装されたROS2ノードファイルです。

ピンホールモデル(Pinhole Model)の適用: 一般的なBEV（鳥瞰変換）ではなく、カメラの幾何学的特性を考慮したピンホールモデルを採用し、ピクセルデータを実世界座標系へ正確に変換するアルゴリズムを設計しました。

2. データ収集の自動化 (checkerboard/)
手動データ収集の非効率を解決するために独自開発したツールです。   

論理的自動化: チェッカーボードの特性を利用し、基準点一点の測定のみで全座標を自動生成。OpenCVで対応ピクセルを抽出・マッピングするプロセスを自動化しました。

3. パラメータ最適化ツール (opencv_test/)
多様な走行環境（照明変化やノイズ）に柔軟に対応するためのチュー닝ツールです。

即時フィードバック: OpenCVのTrackbarを実装し、各種フィルタ値や明暗設定をドラッグ操作でリアルタイムに調整・確認できるようにしました。

4. 座標補正行列の演算 (calibration/)
MATLABを活用し、収集されたデータからピクセル補正行列を算出するアルゴリズムファイルです。
